# Forgejo + Runner Docker Setup

Complete Docker Compose configuration for running Forgejo (Git service) with integrated CI/CD Runner.

## 📋 Overview

This setup provides:
- **Forgejo**: Self-hosted Git service with web UI
- **Docker-in-Docker**: Enables Docker builds in CI/CD pipelines
- **Runner**: Forgejo CI/CD runner for executing workflows
- **Shared Network**: Internal Docker network for service communication

## 🚀 Quick Start

### Prerequisites
- Docker and Docker Compose installed
- Python 3.7+ (for secret generation)

### Setup

1. **Generate secrets** (if not already done):
   ```bash
   python3 secret.py
   ```
   This creates a `.env` file with secure tokens and UUIDs.

2. **Start services**:
   ```bash
   docker-compose up -d
   ```
   Or use the convenience script:
   ```bash
   chmod +x start.sh
   ./start.sh
   ```

3. **Access Forgejo**:
   - Web UI: http://localhost:3000
   - SSH: ssh://git@localhost:222

## 📁 Directory Structure

```
├── compose.yml           # Main Docker Compose orchestration
├── dockerfile           # Custom Docker image
├── requirements.txt     # Python dependencies
├── secret.py           # Secret generation script
├── start.sh            # Startup script
├── .env                # Generated secrets (gitignored)
├── .env.example        # Example environment variables
└── data/               # Persistent volumes
```

## 🔧 Services

### Forgejo
- **Image**: codeberg.org/forgejo/forgejo:latest
- **Port**: 3000 (HTTP), 222 (SSH)
- **Volumes**: forgejo-data
- **Network**: forgejo-network
- **Health Check**: API endpoint verification

### Docker-in-Docker
- **Image**: docker:dind
- **Purpose**: Enables Docker builds inside runner
- **Port**: 2375 (internal TCP)
- **Privileged**: Yes
- **Network**: forgejo-network

### Runner
- **Image**: code.forgejo.org/forgejo/runner:latest
- **Purpose**: CI/CD pipeline execution
- **Volumes**: runner-data, Docker socket
- **Environment**: DOCKER_HOST configuration
- **Network**: forgejo-network
- **Dependencies**: Waits for Forgejo and Docker-in-Docker to be healthy

## 📝 Usage

### View Logs
```bash
# All services
docker-compose logs -f

# Specific service
docker-compose logs -f forgejo
docker-compose logs -f runner
docker-compose logs -f docker-in-docker
```

### Stop Services
```bash
docker-compose down
```

### Restart Services
```bash
docker-compose restart
```

### Remove Everything (including volumes)
```bash
docker-compose down -v
```

## 🔐 Secrets Management

Secrets are automatically generated by `secret.py` and stored in `.env`:
- **FORGEJO_TOKEN**: Secure token for Forgejo
- **FORGEJO_SECRET**: Secret key for session management
- **RUNNER_TOKEN**: Token for runner authentication
- **UUIDs**: Unique identifiers for services

**⚠️ Important**: Never commit `.env` to version control. Use `.env.example` as template.

## 🌐 Networking

All services communicate over an internal `forgejo-network` bridge:
- Forgejo: `http://forgejo:3000`
- Docker: `tcp://docker-in-docker:2375`
- Runner connects to Forgejo via internal network

## 🐛 Troubleshooting

### Runner not connecting to Forgejo
- Verify health checks: `docker-compose ps`
- Check logs: `docker-compose logs runner`
- Ensure internal hostname `forgejo` is used in runner config

### Docker build failures in runner
- Check docker-in-docker logs: `docker-compose logs docker-in-docker`
- Verify privileged mode is enabled
- Ensure sufficient disk space

### Port conflicts
- Change port mappings in `compose.yml` (e.g., `3001:3000`)
- Don't forget to update configuration references
